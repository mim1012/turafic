# 네이버 쇼핑 트래픽 테스트 종합 계획

## 🎯 테스트 목표

**핵심 질문:**
1. 어떤 트래픽 패턴이 순위 상승에 가장 효과적인가?
2. 모바일 vs PC 트래픽의 효과 차이는?
3. 트래픽 양 대비 순위 상승률은?
4. 카테고리별로 효과가 다른가?

---

## 📊 테스트 매트릭스 설계

### A. 테스트 변수 (Independent Variables)

#### 1. 플랫폼 타입
- **모바일 (m.naver.com)**: 70%
- **PC (naver.com)**: 30%
- **모바일 전용**: 100%
- **PC 전용**: 100%

#### 2. 검색 경로
- **통합검색 → 쇼핑탭**: 50%
- **쇼핑 직접검색**: 30%
- **블로그 유입**: 10%
- **카페 유입**: 10%

#### 3. 사용자 행동 패턴
- **빠른 이탈** (30초 이하): 20%
- **일반 둘러보기** (30-90초): 50%
- **심층 탐색** (90-180초): 30%

#### 4. 액션 타입
- **단순 조회**: 40%
- **스크롤 + 리뷰 확인**: 30%
- **옵션 선택**: 20%
- **장바구니 담기**: 10%

#### 5. 트래픽 양
- **소량** (10회): 순위 변동 감지
- **중량** (50회): 효과 검증
- **대량** (100회): 최대 효과 측정

### B. 측정 지표 (Dependent Variables)

#### 1. 순위 변동
- **절대 순위**: 1~100위
- **순위 변화량**: -50 ~ +50위
- **페이지 이동**: 페이지 간 이동 여부

#### 2. 효과성 지표
- **순위 상승률**: 상승 건수 / 전체 건수
- **평균 상승폭**: 평균 몇 위 상승
- **최대 상승폭**: 최대 몇 위까지 상승

#### 3. 안정성 지표
- **봇 탐지율**: 차단/실패 건수
- **순위 유지율**: 상승 후 며칠 유지
- **역효과율**: 순위 하락 건수

## 🔬 실험 통제 요소

### 1. 상수로 유지할 변수
- 테스트 시간대: 오전 10시~오후 6시 (트래픽 피크 시간 회피)
- 요일: 화~목 (주말/월요일 제외)
- 순위 체크 간격: Before → 트래픽 → 30분 대기 → After
- IP 로테이션: 매 요청마다
- User-Agent: 실제 데이터 기반 풀 사용

### 2. 랜덤화할 변수
- 체류 시간: ±20% 범위 내 랜덤
- 스크롤 횟수: ±1회 랜덤
- 액션 발생 확률: 설정값 ±10%
- 대기 시간: 정규분포 (μ=150초, σ=30초)

### 3. 기록할 메타 데이터
- 테스트 시작/종료 시간
- 네트워크 상태 (요청 성공률)
- 경쟁사 변동 (같은 키워드 내 다른 상품 순위)
- 네이버 알고리즘 업데이트 여부
