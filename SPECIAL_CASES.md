# 특수 테스트 케이스 (Advanced Scenarios)

## 문제점: 일반적인 케이스는 효과 없음
```
❌ 이미 시도된 일반 케이스:
  - 단순 클릭
  - 짧은 체류
  - 장바구니 담기만
  - 리뷰 클릭만

→ 네이버가 이미 필터링하고 있을 가능성 높음
```

## ✨ 특수 케이스 전략

---

## CASE_S1: 구매 완료 시뮬레이션 ⭐⭐⭐⭐⭐
**가설**: 실제 구매까지 진행하는 트래픽이 가장 강력한 신호

### 시나리오
```
1. 네이버 검색 → 쇼핑탭
2. 여러 상품 비교 (2-3개 클릭) ← 차별화
3. 타겟 상품 선택
4. 상세 정보 꼼꼼히 확인 (스크롤 깊게)
5. 리뷰 읽기 (2-3개)
6. 옵션 선택 (색상, 사이즈 등)
7. 장바구니 담기
8. 장바구니 이동
9. 주문서 작성 페이지 진입 ← 핵심!
10. (실제 결제는 안 함, 여기서 취소)
체류: 120-180초 (2-3분)
```

### 차별점
- **구매 퍼널 끝까지** 진행
- 네이버 페이: "주문서 작성" 페이지까지 접근
- 실제 구매 의사가 매우 강한 신호

---

## CASE_S2: 재방문 패턴 (찜 → 재접근) ⭐⭐⭐⭐⭐
**가설**: 여러 번 방문하는 사용자 = 구매 가능성 높음

### 시나리오 (2회 방문)
```
【1차 방문】
1. 상품 발견
2. 상세 정보 확인
3. 찜하기(Keep) 클릭 ← 핵심!
4. 다른 상품도 비교
5. 나가기
체류: 60-90초

【30분-1시간 후 2차 방문】
1. 네이버 쇼핑 → 찜 목록 접근
2. 찜한 상품 재클릭
3. 리뷰 다시 확인
4. 가격 변동 확인
5. 장바구니 담기 또는 구매
체류: 90-120초

총 2회 세션 = 강력한 관심 신호
```

### 차별점
- **재방문 패턴** (Keep 사용)
- 시간차 두고 접근
- 구매 고민하는 실제 사용자 패턴

---

## CASE_S3: 비교 쇼핑 후 최종 선택 ⭐⭐⭐⭐
**가설**: 경쟁 상품과 비교 후 선택 = 합리적 구매자

### 시나리오
```
1. 키워드 검색
2. 상품 A 클릭 (경쟁사)
   - 간단히 확인 (30초)
   - 뒤로가기
3. 상품 B 클릭 (경쟁사)
   - 간단히 확인 (30초)
   - 뒤로가기
4. **타겟 상품** 클릭 ← 진짜 목표
   - 상세 확인 (리뷰, 가격, 옵션)
   - 장바구니 또는 구매
   - 체류: 90-120초
5. (선택적) 다시 경쟁사 확인 후 재방문

총 체류: 150-210초 (여러 상품 비교)
```

### 차별점
- **3개 이상 상품 비교**
- 타겟 상품에 가장 오래 체류
- 네이버: "이 사용자가 이 상품을 선택했구나" 신호

---

## CASE_S4: 외부 유입 시뮬레이션 ⭐⭐⭐⭐
**가설**: 블로그/SNS 등 외부에서 유입된 트래픽이 가치 높음

### 시나리오
```
1. **Referrer 조작**: 구글 검색 or 네이버 블로그에서 유입된 것처럼
   - URL에 utm_source 파라미터 추가
   - 예: https://shopping.naver.com/products/12345?utm_source=blog

2. 직접 상품 URL 접근 (검색 건너뛰기)
3. 꼼꼼히 확인 (블로그 후기 보고 온 사용자처럼)
4. 장바구니 담기
체류: 90-120초
```

### 차별점
- **외부 유입** 트래픽 (검색이 아님)
- Referrer/utm 파라미터로 출처 표시
- 블로그 리뷰 보고 온 사용자 패턴

---

## CASE_S5: 모바일 앱 + 웹 크로스 플랫폼 ⭐⭐⭐
**가설**: 여러 플랫폼에서 접근 = 관심도 높음

### 시나리오
```
【모바일 앱】
1. 네이버 쇼핑 앱 실행
2. 상품 검색 및 클릭
3. 찜하기
4. 앱 종료

【30분 후 모바일 웹】
1. 모바일 Chrome에서 네이버 쇼핑
2. 찜 목록 확인
3. 장바구니 담기

→ 네이버: "여러 플랫폼에서 접근하는 진지한 구매자"
```

### 차별점
- **크로스 디바이스/플랫폼**
- 앱 + 웹 통합 추적
- 복잡하지만 효과 높을 가능성

---

## CASE_S6: 타임딜/이벤트 참여 ⭐⭐⭐
**가설**: 이벤트 참여자 = 활성 사용자

### 시나리오
```
1. 타임딜/쿠폰 페이지 접근
2. 쿠폰 받기 클릭
3. 쿠폰 적용된 상품 검색
4. 타겟 상품에 쿠폰 적용
5. 장바구니 담기 (쿠폰 적용 상태)
6. 할인 금액 확인
체류: 120-150초
```

### 차별점
- **쿠폰/이벤트 활용**
- 가격 민감 구매자 패턴
- 실제 구매 가능성 높은 신호

---

## CASE_S7: 소셜 활동 (공유/리뷰 작성 의도) ⭐⭐⭐⭐
**가설**: 공유하거나 리뷰 쓰려는 사용자 = 충성도 높음

### 시나리오
```
1. 상품 상세 페이지 접근
2. 리뷰 섹션 정독 (스크롤 깊게)
3. 도움이 돼요 클릭 (다른 사람 리뷰에)
4. 공유하기 버튼 클릭 ← 핵심!
   - 카카오톡 공유 or URL 복사
5. 장바구니 담기
체류: 90-120초
```

### 차별점
- **소셜 액션** (공유, 도움돼요)
- 네이버: "이 상품을 추천하는 사용자"
- 바이럴 효과 신호

---

## CASE_S8: 셀러 스토어 탐색 ⭐⭐⭐
**가설**: 브랜드/셀러에 관심 = 충성 고객

### 시나리오
```
1. 상품 상세 페이지
2. 판매자 정보 클릭
3. 셀러 스토어 페이지 이동
4. 셀러의 다른 상품 2-3개 확인
5. 원래 상품으로 복귀
6. 장바구니 담기
체류: 120-180초 (셀러 탐색 포함)
```

### 차별점
- **브랜드 충성도** 신호
- 셀러 전체 상품 관심
- 재구매 가능성 높은 사용자

---

## CASE_S9: Q&A 작성 시도 ⭐⭐⭐⭐
**가설**: 문의하는 사용자 = 구매 직전

### 시나리오
```
1. 상품 상세 페이지
2. 리뷰 읽기
3. Q&A 섹션 확인
4. "문의하기" 버튼 클릭
5. 문의 작성 페이지 진입
6. (실제 작성은 안 함, 로그인 필요 시 취소)
7. 장바구니 담기
체류: 90-120초
```

### 차별점
- **구매 전 문의** (구매 의사 매우 강함)
- 문의 작성 페이지까지 접근
- 진지한 고민 신호

---

## CASE_S10: 가격 알림 설정 ⭐⭐⭐
**가설**: 가격 추적 = 장기 구매 의사

### 시나리오
```
1. 상품 상세 페이지
2. 가격 비교 확인
3. "가격 알림 받기" 클릭
4. (로그인 필요 시 취소)
5. 찜하기
6. 다른 옵션 확인
체류: 60-90초
```

### 차별점
- **가격 추적** 설정
- 장기 관심 고객
- 할인 시 구매 의도

## 프로토타입 수정 제안

### AS-IS (기존)
```python
# 너무 단순
1. 상품 URL 열기
2. 스크롤
3. 장바구니 클릭
4. 대기
```

### TO-BE (특수 케이스)
```python
# CASE_S3: 비교 쇼핑 후 선택

1. 검색 결과 페이지 열기
2. 경쟁 상품 1 클릭 → 30초 체류 → 뒤로가기
3. 경쟁 상품 2 클릭 → 30초 체류 → 뒤로가기
4. **타겟 상품** 클릭
5. 리뷰까지 깊게 스크롤
6. 옵션 선택
7. 장바구니 담기
8. 90-120초 체류

총 시간: 180초 (3분)
효과: 기본 케이스 대비 3-5배 예상
```
